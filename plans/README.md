# plans/

Aktive Mehr-Schritt-Pläne. Hält den **Umsetzungsstand** sichtbar: was ist getan,
was offen. Ein Plan wird **gelöscht**, wenn die Arbeit durch ist (das Ergebnis lebt
dann in Spec/Code/Docs/Journal). Offene Punkte, die einen Merge überdauern, werden
**GitHub-Issues** (SPEC-repo-conventions §4).
