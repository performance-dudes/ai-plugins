# journal/

The project's running logbook — a short, honest note **per PR**: what was done,
what came out of it, and what was learned. It is a technical exchange between
teammates (people and agents), not a changelog. Newest first.

Follows the internal `knowledge/journal` skill. One file per entry:
`journal/<YYYY-MM-DD>-<slug>.md`.
