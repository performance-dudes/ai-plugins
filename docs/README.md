# docs/

Erklärung der Komponenten — **wie** etwas funktioniert/zu benutzen ist (nicht das
Intent, das steht in `specs/`). Je Plugin ein Unterordner `docs/<plugin>/` (eine
Seite pro Komponente); Übergreifendes an der `docs/`-Wurzel.

**Nie** Specs hier ablegen (siehe [`specs/`](../specs/README.md)). Doku liegt
bewusst **außerhalb** der Plugin-Ordner: eine `.md` in `commands/`/`agents/` würde
als echtes Command/Agent geladen.
